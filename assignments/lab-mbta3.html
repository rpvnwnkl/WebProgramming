Instructions

Last week, I gave you a URL to the JSON real-time data feed of MBTA Red Line subway trains: https://defense-in-derpth.herokuapp.com/redline.json. This week, I am deleting what I provided that serves the JSON data feed. Therefore, your map will no longer work. To make your map work again, you must recreate the server-side.

I can tell you how my JSON data feed actually worked: it pulls the data from http://developer.mbta.com/lib/rthr/red.json, the original source. In other words, what you used in the last lab was a mirror of what is provided by the MBTA. However, I added one thing that allowed one to directly access my JSON data feed from client-side JavaScript. You see, if you simply change https://defense-in-derpth.herokuapp.com/redline.json to http://developer.mbta.com/lib/rthr/red.json in your last lab, your map will still not work because of a fundamental security principle in client-side JavaScript.

Basic Requirements of the Server

You will recreate the server that will serve the real-time schedule JSON feed MBTA Red Line subway trains. Basic requirements:

  1. The server shall be written in Node.js and using the Express web application framework. You are allowed to use Node packages.
  2. Your work must be deployed onto Heroku.
  3. Your server must have one route: GET /redline.json. Accessing this route shall return a JSON string, the contents of the original source of the JSON data feed http://developer.mbta.com/lib/rthr/red.json.
  4. Cross-Origin Resource Sharing must be enabled for GET /redline.json.

Once you have completed the basic requirements of the server, all you need to do is to change the URL https://defense-in-derpth.herokuapp.com/redline.json to https://NAMEOFYOURHEROKUAPP.herokuapp.com/redline.json in your map and your map should work again. If your map does not work, or if JavaScript changes are required to your previous work, then something is wrong with your server.

Going Beyond

  1. Record the latitude and longitude of users accessing your map in MongoDB. Please DO NOT record more information than that (e.g., IP address) for privacy purposes. You will need to implement a new route on the server (using HTTP POST).
  2. Keep track of the most popular Red Line stations clicked on. Each time when a user clicks on a train station, record the information into a database (e.g., Mongo). You will need to implement a new route on the server (using HTTP POST).
  3. The MBTA JSON feed http://developer.mbta.com/lib/rthr/red.json is actually deprecated, but has not been fully decommissioned (think of reasons why full decommission is a bad idea). Upgrade to the newest version of the MBTA Real-Time API. See http://www.mbta.com/rider_tools/developers/ for more details.
  4. Build another web application + server with the same basic requirements except using a different programming language and framework (e.g., using Django or Flask for Python, Rails or Sinatra for Ruby). Assess the differences and similarities in your README.
